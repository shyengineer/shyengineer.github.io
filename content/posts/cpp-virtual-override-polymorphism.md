## 1. 자! '가상 함수(virtual)'가 왜 필요할까요?

지난 시간에는 상속을 배웠지요? 하지만 단순히 물려받는 것만으로는 부족합니다!! 부모님이 "공부해라!"라고 말씀하셔도, 자식마다 "수학 공부", "음악 공부" 등 구체적으로 하는 일은 다를 수 있잖아요!!

이처럼 **부모 클래스에서 이름을 정해주고, 실제 행동은 자식 클래스에서 결정하고 싶을 때** 사용하는 것이 바로 `virtual` 키워드입니다!!

---

## 2. virtual: "나중에 자식이 정할 거야!!"

부모 클래스의 메서드 앞에 `virtual`을 붙이면, C++ 시스템은 "아, 이 함수는 나중에 자식이 바꿀 수도 있겠구나!"라고 기억해 둡니다!!

```cpp
class Parent {
public:
    virtual void study() {
        std::cout << "기본 공부를 합니다!!" << std::endl;
    }
};
```
이렇게 virtual을 붙여줘야만, 나중에 우리가 다형성을 사용할 때 자식의 함수가 올바르게 호출된답니다!! 정말 중요하겠죠?

## 3. override: "제가 확실히 물려받아 고쳤어요!!"
자! 자식 클래스에서 부모의 함수를 새로 만들 때는 override라는 꼬리표를 붙여주는 것이 매너입니다!!

```C++

class Child : public Parent {
public:
    void study() override { // 부모의 study를 재정의했다는 표시!!
        std::cout << "물리 공부를 열정적으로 합니다!!" << std::endl;
    }
};
```
왜 override를 써야 할까요?
실수 방지!!: 부모 함수의 이름을 살짝 틀리게 적으면, 시스템이 "어? 부모한테 그런 함수 없는데?"라고 바로 알려줍니다!!

가독성 짱!!: 코드를 읽는 다른 사람이 "아, 이건 부모 함수를 바꾼 거구나!"라고 바로 알 수 있죠!! 쉽죠?

## 4. 왜 팩토리 패턴 전에 이걸 배워야 할까요?
자! 우리가 나중에 배울 팩토리 패턴은 "어떤 자식 객체를 만들지"를 결정하는 공장입니다!!

이 공장에서 물건을 찍어낸 뒤에, 부모 타입의 포인터로 자식 객체들을 자유자재로 다루려면 오늘 배운 virtual과 override가 필수랍니다!! 이 개념이 흔들리면 공장이 제대로 돌아가지 않겠죠?

## 5. 요약하자면 이렇습니다!!
virtual: 부모가 자식에게 "너만의 스타일로 바꿔도 돼!!"라고 허락해 주는 키워드!!

override: 자식이 부모에게 "제가 멋지게 바꿨습니다!!"라고 보고하는 꼬리표!!
